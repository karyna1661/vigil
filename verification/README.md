# Verification

Store proof artifacts here.

Examples:

- command output captures
- screenshots
- structured logs
- benchmark results
- before/after comparisons
- manual verification notes

Recommended layout:

- `verification/phase-0-proof/`
- `verification/phase-1-proof/`
- `verification/phase-2-proof/`

Tests are not the only proof.

If the project has operator-facing behavior, keep direct proof of that behavior here.
